ᛏ TypeScript · CLI · Dependency Injection

Un contexto para gobernarlos a todos

Tyr es un framework CLI TypeScript-native para construir, ejecutar y componer herramientas de automatización — sin boilerplate. Escribe una función, tyr se encarga del resto: routing de argumentos, inyección de dependencias, formato de errores y documentación viva.

ᛏ El problema

Basta de scripts dispersos

Tyr resuelve el problema de fragmentación en el scripting de DevOps. En lugar de mantener shell scripts sueltos, utilidades de Node improvisadas y automatizaciones sin documentar, Tyr te da un único contexto tipado e inyectado por dependencias desde el que opera cada comando.

ᛏ Quick Start

De cero a comando en tres pasos

Sin configuración previa. Sin ceremonia. Solo escribe la lógica.

1 Crear el comando
terminal 
tyr gen greet greet

Genera src/commands/greet.tyr.ts y lo registra en config/map.yml.

2 Editar el comando
greet.tyr.ts 
import { TyrContext } from '../core/Kernel';

export default ({ task, fail, logger }: TyrContext) => {
    return async (args: string[]) => {
        if (args.length === 0) {
            fail('A name is required', 'Usage: tyr greet <name>');
        }
        await task('Greeting', async () => {
            logger.success(`Hello, ${args[0]}!`);
        });
    };
};
3 Ejecutarlo
terminal 
tyr greet World
→ ✔ Hello, World!

Los comandos se registran automáticamente y quedan disponibles como sub-comandos de tyr.

ᛏ Arquitectura

Conceptos clave

Cuatro piezas que explican todo el framework.

01 · Kernel

El motor de ejecución

El Kernel arranca en cada invocación, carga config/map.yml, resuelve el nombre del comando a un módulo y lo ejecuta con un TyrContext completamente montado.

Nunca lo instancias tú mismo — corre automáticamente vía bin/tyr.js.

flujo
tyr deploy --force
   │
   ▼
Kernel.run('deploy', ['--force'])
   │  lee config/map.yml
   ▼
Container.resolve() // construye TyrContext
   │
   ▼
deploy.tyr.ts(ctx)(args)
02 · Container & DI

Todos los Managers, un único punto

El Container instancia cada Manager una sola vez y lo inyecta en cada comando a través del TyrContext. Nunca importas un Manager directamente — lo desestructuras del contexto.

deploy.tyr.ts
export default ({ shell, fs, git, logger }: TyrContext) => {
    return async (args: string[]) => {
        const branch = await git.currentBranch();
        logger.info(`On branch: ${branch}`);
    };
};
03 · TyrContext

Tu API de superpoderes

El objeto inyectado en cada comando. Expone todos los Managers disponibles más utilidades del framework:

  • logger — salida coloreada por terminal
  • task(label, fn) — envuelve operaciones async con contexto de error
  • fail(msg, suggestion?) — lanza un TyrError controlado
  • run(cmd, args) — invoca otro comando Tyr programáticamente
  • shell, fs, git, docker, sql, web… — todos los Managers
error handling
// controlado: validación esperada
fail('Config file not found', 'Create a config.yml.');

// no controlado: se envuelve y se formatea
await task('Connecting to database', async () => {
    await sql.connect(connectionString);
});

// stack trace completo
// $ tyr deploy --debug
04 · Commands

Composición sin fricción

Un comando es una función exportada por defecto que recibe un TyrContext y devuelve un handler async. Se registran en config/map.yml y pueden invocarse entre sí con run(), habilitando pipelines completos a partir de piezas pequeñas.

config/map.yml
commands:
  greet: ./src/commands/greet.tyr.ts
  deploy: ./src/commands/deploy.tyr.ts
  rollback: ./src/commands/rollback.tyr.ts
ᛏ Ejemplos

Tyr en acción

De un saludo a un agente de IA que edita tu código — mismo patrón, mismo contexto.

import { TyrContext } from '../core/Kernel';

// Compón varios Managers en un único pipeline, con manejo de errores automático.
export default ({ task, fail, git, docker, shell, logger }: TyrContext) => {
    return async (args: string[]) => {
        const branch = await git.currentBranch();
        if (branch !== 'main' && !args.includes('--force')) {
            fail(`Refusing to deploy from '${branch}'`, 'Use --force to override.');
        }

        await task('Building image', async () => {
            await docker.compose.up({ build: true, detach: true });
        });

        await task('Running migrations', async () => {
            await shell.exec('npm run migrate');
        });

        logger.success(`Deployed from ${branch} 🚀`);
    };
};
// AIContextManager orquesta el loop de la IA: explora el proyecto, aplica
// cambios como bloques Search/Replace y valida el resultado en un sandbox.
export default ({ aiContext, prompts, tokens, logger }: TyrContext) => {
    return async (args: string[]) => {
        const tree = await aiContext.buildExplorationTree(process.cwd());
        const messages = await prompts.build('ai-code', { task: args.join(' '), tree }, process.cwd());

        const result = await aiContext.runCodeAgent(process.cwd(), messages, tokens, {
            priority: 'media-prioridad',
        });

        logger.success(`${result.filesChanged.length} archivo(s) modificados`);
    };
};
// ChatManager expone una app de chat local con panel de archivos.
export default ({ chat, aiVendor, logger }: TyrContext) => {
    return async (args: string[]) => {
        chat.onMessage(async ({ message, history, dir }) => {
            const result = await aiVendor.complete([{ role: 'user', content: message.text }]);
            return result.content;
        });

        const session = await chat.open(args[0] ?? process.cwd(), { splitRatio: 0.4 });
        logger.success(`Chat ready at: ${session.url}`);
    };
};

// $ tyr chat ./mi-proyecto --split 0.35
// → Chat ready at: http://localhost:4646
# Instala comandos de terceros a partir de un manifest.json remoto
tyr --add https://raw.githubusercontent.com/someuser/tyr-modules/main/manifest.json my-module

tyr --modules            # lista módulos registrados
tyr --modules sync       # reinstala lo que falte
tyr --update              # git pull + refresco forzado
tyr --del my-module       # desinstala el módulo por completo

# Publica tu propia colección de comandos

tyr --manifest             # genera ~/.tyr/manifest.json
ᛏ Módulos destacados

Un ecosistema de Managers listo para usar

19 Managers cubren desde shell y git hasta bases de datos y agentes de IA. Estos son los más interesantes.

AIContextManager

Orquesta el agente de código: explora el proyecto, aplica cambios como parches Search/Replace bajo un guard anti-sobrescritura y valida el resultado en sandbox, con auto-reintento si algo falla.

AIVendorManager

Abstrae Anthropic, OpenAI y Gemini bajo una interfaz normalizada de mensajes, bloques de contenido y tool-use. Cambia de vendor sin tocar tu lógica de comando.

ChatManager

Levanta una app web local con chat y explorador de archivos. Adjunta imágenes, redimensiona paneles y conecta tu propio handler de respuestas.

SandboxManager

Ejecuta y valida cambios de forma aislada antes de darlos por buenos — la red de seguridad que usa el agente de IA antes de tocar tus archivos de verdad.

ShellManagershell

Ejecuta comandos del sistema y mantiene el CWD para encadenar operaciones en carpetas concretas.

FileSystemManagerfs

Capa de abstracción sobre el filesystem, con backups automáticos al sobrescribir y escrituras idempotentes.

GitManagergit

Wrapper de operaciones Git comunes: init, commits, clonado y consulta de rama actual.

SystemManagersystem

Utilidades de mantenimiento del SO, como liberar un puerto ocupado (killPort).

PackageManagerpkg

Gestor de paquetes agnóstico al SO: detecta apt, brew o dnf e instala software nativo.

WorkspaceManagerworkspace

Gestiona directorios de trabajo locales: detecta repos existentes, crea ramas y abre el proyecto en VSCode.

SetupManagersetup

Bootstrap de entornos de desarrollo: detección de SO, dependencias, docker-compose y generación de Makefiles.

DockerManagerdocker

Manager de bajo nivel sobre el Docker Daemon: arranca contenedores, comprueba estados y gestiona stacks de Compose.

WebManagerweb

Peticiones HTTP y scraping con cheerio, listo para consumir APIs o extraer datos de páginas.

JiraManagerjira

Integración con la API REST de Jira: trae y selecciona issues asignadas, con fallback manual si Jira no está disponible.

SQLManagersql

Conector para SQL Server (MSSQL) con gestión de pool de conexiones.

MongoManagermongo

Conector MongoDB que gestiona el ciclo de vida de la conexión y expone un CRUD genérico.

MemoryManagermemory

Memoria persistente de conversaciones de IA por proyecto — solo gasta tokens de contexto cuando de verdad ayuda.

TokenManagertokens

Estima el coste en tokens de un prompt antes de enviarlo y registra el uso real una vez completada la petición.

PromptTemplateManagerprompts

Plantillas de sistema/usuario reutilizables para instrucciones de IA uniformes en todo el framework.

ᛏ Ecosistema abierto

Instala y publica comandos como módulos

Cualquiera puede distribuir comandos como un manifest.json — un mapa plano de nombre de comando a la URL raw de su archivo .tyr.ts en GitHub. Tyr los instala, los actualiza y los desinstala sin tocar tus comandos escritos a mano.

  • Registra un módulo con tyr --add <url-manifest> [nombre]
  • Sincroniza todos con tyr --modules sync
  • Fuerza refresco con tyr --update
  • Desinstala limpio con tyr --del <nombre>
manifest.json 
{
  "deploy": "https://raw.githubusercontent.com/someuser/tyr-modules/main/deploy.tyr.ts",
  "rollback": "https://raw.githubusercontent.com/someuser/tyr-modules/main/rollback.tyr.ts",
  "$env": "https://raw.githubusercontent.com/someuser/tyr-modules/main/.env.example"
}
